Tracking Management Analysis

What is Session Tracing?

Session tracing is a mechanism that records every interaction, API call, and state change within the Pi coding agent. Each event is timestamped and linked to its parent event, creating a complete audit trail of the execution flow.

How Does It Work?

Event Recording:

  • Every action is recorded as a JSON event with a unique ID
  • Parent-child relationships are maintained via parentId references
  • Events include timestamps, metadata, and execution details
  • Complete execution chains can be reconstructed from the traces

Key Features

  • Complete Audit Trail: Every action recorded with full context
  • Cost Transparency: Track API tokens and costs with precision
  • Performance Analytics: Analyze execution patterns and identify bottlenecks
  • Security & Compliance: Maintain complete logs for compliance and audits
  • Tool Integration: Track bash, read, write, edit operations
  • Parent-Child Chain: Follow execution hierarchy for debugging

System Documentation

Data Storage Format

Sessions are stored as JSONL (JSON Lines) files. Each line is a complete JSON object representing one event:

{"type":"session","id":"abc123","timestamp":"2026-02-19T...","cwd":"C:\\Users\\bhavy"} {"type":"model_change","id":"def456","parentId":"abc123","provider":"anthropic"} {"type":"message","id":"ghi789","parentId":"def456","message":{"role":"user",...}}

Parent-Child Execution Chain

Events are linked hierarchically showing execution flow:

Session Start ├─ model_change: Select Claude Haiku ├─ thinking_level: off ├─ message: User asks question ├─ message: AI responds with tool calls ├─ message: Tool result (bash output) └─ message: Final AI response

System Architecture

The tracing system works as follows:

  • Pi Agent executes user commands
  • Session Manager creates and tracks events
  • Event Recorder logs all actions with metadata
  • Parent-child relationships are maintained
  • JSONL files store events in append-only format
  • Timestamps enable timing analysis
  • Complete audit trail is created
