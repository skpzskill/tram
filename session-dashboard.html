Tracking Management Analysis

1. What is Session Tracing?

Session tracing is a mechanism that records every interaction, API call, and state change within the Pi coding agent. Each event is timestamped and linked to its parent event, creating a complete audit trail of the execution flow.

2. How Does It Work?

üìå Event Recording:

  • Every action is recorded as a JSON event with a unique ID
  • Parent-child relationships are maintained via parentId references
  • Events include timestamps, metadata, and execution details

üîó Parent-Child Chain:

  • Session initialization (root event with no parent)
  • Model selection and configuration
  • User message ‚Üí Assistant processing ‚Üí Tool calls
  • Tool execution and results
  • Response generation with metrics

3. Event Types

Event Type Description Purpose
session Session initialization Marks the start of a session with metadata
model_change Model/provider selection Tracks which AI model is being used
message User/assistant messages Records conversations and API interactions
thinking_level_change Extended thinking mode Tracks when deep analysis mode is toggled
tool_result Tool execution results Captures output from bash, read, write operations

4. Tracking Management Features

üìä Token & Cost Tracking:

  • Input tokens, output tokens, and total tokens per request
  • Cache read/write metrics for optimization
  • API cost calculation in real-time

üîç Execution Flow Tracking:

  • Parent-child event relationships
  • Sequential timestamps for timing analysis
  • Tool call attribution and result mapping

5. Data Storage Format

Sessions are stored as JSONL (JSON Lines) files in: ~/.pi/agent/sessions/

Each line is a complete JSON object representing one event. This format allows for:

  • Streaming processing without loading entire file
  • Easy appending of new events
  • Efficient storage and retrieval

6. Use Cases & Applications

üîê Audit & Compliance: Complete record of all actions for security audits

üìà Performance Analysis: Identify bottlenecks and optimize tool usage

üí∞ Cost Management: Track API spending and identify cost optimization opportunities

üêõ Debugging: Trace execution flow to identify issues

üìä Usage Analytics: Understand how the agent is being used
